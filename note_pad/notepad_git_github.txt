#github 인증하는 방법

1. id, pw로 브라우저 인증 -> 취미 생활

2. HTTPS + Persnal access token -> 1번보다 약간 고급진 방법

3. SSH key -> 가장 안전하고, 안정적인 방법(1대의 컴퓨터만 사용가능)

------------------------------------------------------------------------
------------------------------------------------------------------------
# SSH key 방법

1. 제어판-> 사용자 계정-> window 자격 증명 관리자-> github 브라우저 인증 [제거]

2. '돋보기'에 'cmd' 검색, 열어서 "mkdir %USERPROFILE%\.ssh\git_keys"입력 및 enter

3. c드라이브-> 사용자(or, user)-> ".ssh" 및 기타 폴더 확인하기

4. 'cmd'에서 "ssh-keygen -t ed25519 -C "ho8566z@gmail.com"
 -f %USERPROFILE%\.ssh\git_keys\id_ed25519_main" 입력하기

5. .ssh-> git_keys에서 개인키, 공개키 확인하기

6. github-> user.settings-> [SSH and GPG keys]에 새로운 SSH key 등록하기
(cmd에서 "type %USERPROFILE%\.ssh\git_keys\id_ed25519_main.pub" 입력해서 
 SSH key 확인)

7. '돋보기'에 "services.msc"를 입력하고, 'openssh agent~'를 찾아서 '사용안함'을
'자동'으로 [선택]하고, [적용], [시작], [확인]하기

8. 'cmd'에서 "ssh-add %USERPROFILE%\.ssh\git_keys\id_ed25519_main" 입력하기

9. 등록 완료 확인하고, "ssh -T git@github.com" 입력하고, "yes"입력 및 enter

10. 연결 상태 확인하고 메세지 확인하기

11. 각각의 'Local Repository'에서 'git bash' 실행해서 github에 인식시키기

Git 브랜치 전략

main : 배포용 브랜치 (항상 안정 상태 유지)
dev : 개발 통합 브랜치
feature/* : 기능 단위 작업 브랜치
(feature/*_ex: feature/member, feature/bank, feature/memo 등등)

------------------------------------------------------------------------
(각각)개발자들 --> main --> server

(각각)개발자들 --> dev --> main --> server

(각각)개발자들 --> feature/* --> PR과정(검증단계) --> dev --> PR과정(검증단계) 
--> main --> server

------------------------------------------------------------------------
# 작업 흐름

1단계: feature 브랜치에서 작업
-git checkout -b feature/member
	실제 기능 개발은 모두 feature/*에서 진행, 작업 후 commit

2단계: dev 최신화
-git checkout dev
-git pull origin dev
	항상 최신 상태 유지 (충돌 방지)

3단계: feature → dev merge & 테스트
-git checkout dev
-git merge feature/member
	로컬에서 먼저 테스트 진행, 문제 없으면 다음 단계 진행

4단계: 원격 저장소로 push
-git push origin feature/member
-git push origin dev

5단계: PR 생성 (feature → dev)
	GitHub에서 Pull Request 생성하고, 코드 리뷰 진행하고, 테스트 통과 후 merge

6단계: dev → main 배포
	dev가 안정화되면 main으로 merge
-git checkout main
-git merge dev
-git push origin main
